User Story: As a field officer, I need to conduct Group Recognition Tests (GRT) at customer meetings to verify group members and approve loan disbursements.

Primary GRT Flow

Screen 1

GRT Main Screen

Officer views center details, meeting schedule, and GRT history. Can filter customers by status (In Progress/On-Hold).

  • Center name and next meeting date
  • GRT history badge shows active GRTs
  • Customer list with avatars and IDs
  • Tab filters: In Progress, On-Hold
  • Start GRT button (disabled until selection)
Screen 2

Expired GRT

Shows when a previous GRT session has expired. Officer can view history or start a new GRT.

  • Warning indicator for expired GRT
  • Time since expiration displayed
  • History access available
  • Can initiate new GRT session
Screen 3

GRT History

Timeline view of all previous GRT sessions with dates, locations, and outcomes.

  • Chronological timeline
  • Meeting location for each session
  • Attendance counts
  • Completion status indicators

Key Features & Components

Component

Customer Selection

Multi-select interface for choosing group members to include in GRT.

  • Checkbox selection per customer
  • Customer avatar, name, and ID
  • Filter by status tabs
  • Bulk selection support
Component

Group Photo Capture

Take mandatory group photo for GRT verification and documentation.

  • Camera interface integration
  • Selected customers shown
  • Photo preview before saving
  • Retake option available
State

Empty States

When no customers are available or no GRT history exists.

  • Friendly illustration
  • "None added yet!" messaging
  • CTA to add members
  • Guidance for next steps

Business Rules

  • Minimum 3 customers required to start GRT
  • GRT expires after 24 hours if not completed
  • Group photo is mandatory for completion
  • Officer can pause GRT and resume later (On-Hold status)
  • History retained for audit and compliance
  • Cannot start new GRT if one is already in progress
